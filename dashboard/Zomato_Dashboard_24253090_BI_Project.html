Predictive Analytics

How well can we predict restaurant ratings?

A Random Forest regression model was trained using location, cuisines, cost, online ordering, table booking and votes to estimate the cleaned numeric rating for each restaurant.

Model summary

  • Target: numeric rating (rate).
  • Features: location, cuisines, votes, approx_cost_for_two, online_order, book_table.
  • Algorithm: Random Forest Regressor with mixed numeric/categorical preprocessing.
  • Performance: R² ≈ 0.67, MAE ≈ 0.14 on the held-out test set.

Insights & Recommendations

What should Zomato do with these findings?

Strategic insights

  • High-density areas (BTM, HSR, Koramangala) are commercially important but highly competitive.
  • Premium localities deliver stronger ratings, suggesting scope for curated “best of” experiences.
  • Engagement (votes) is a strong predictor of rating quality and customer trust.
  • Online ordering and table booking are associated with more stable and slightly higher ratings.

Recommended actions

  • Deepen presence in high-rating zones through loyalty programmes and targeted campaigns.
  • Promote niche cuisines to diversify choice beyond dominant North Indian and Chinese offerings.
  • Incentivise customers to leave votes/reviews to strengthen the rating system.
  • Encourage restaurants with strong potential to adopt online ordering and table booking.
  • Use the predictive model to flag under-exposed but high-potential restaurants for promotion.

Data, Methodology & AI

How was this analysis carried out?

Data & preparation

  • Dataset: Zomato Bangalore Restaurants (Kaggle).
  • Cleaning: removing duplicates, parsing ratings, normalising cost and votes, handling missing values.
  • Feature engineering: cuisine count, popularity score, encoded locations and cuisines.
  • Tools: Python (pandas, seaborn, Plotly, scikit-learn) in a Kaggle/Jupyter environment.

AI usage

Generative AI (ChatGPT) was used to support code generation, debugging, visual design ideas and narrative structuring. All code was executed, tested and adapted to the actual dataset, and analytical decisions were validated against the underlying data.
